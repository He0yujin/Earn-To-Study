# 📚 Earn To Study (E2S) - Group Edition DApp 실행 가이드 (Mac OS 기준)

본 프로젝트는 복잡한 `npm` 설치 환경 구축 없이, 초보자도 맥북(Mac) 환경에서 즉각적으로 해커톤 데모 시연이 가능하도록 설계된 순수 HTML/CSS/JS 기반의 프론트엔드를 제공합니다. 이번 버전에서는 **스터디 그룹을 형성하고 서로의 현황을 파악하는 소셜 기능**이 추가되었습니다!

## 1. 스마트 컨트랙트 배포 (Remix IDE 활용)
1. 웹 브라우저(Chrome 권장)를 열고 [Remix IDE (https://remix.ethereum.org/)](https://remix.ethereum.org/)에 접속합니다.
2. 기존 컨트랙트가 있다면 삭제하고, 로컬에 있는 `contracts/EarnToStudy.sol` 파일의 새 코드를 복사하여 붙여넣습니다.
3. 좌측 3번째 탭 **Solidity Compiler**를 클릭하고, Compiler 버전을 `0.8.20` 이상으로 맞춘 뒤 **Compile EarnToStudy.sol** 버튼을 누릅니다.
4. 좌측 4번째 탭 **Deploy & run transactions**를 선택합니다.
   - 메타마스크를 통한 완벽한 데모를 원한다면 `Environment`를 **Injected Provider - MetaMask**로 변경하세요. (지갑 팝업 승인)
   - 테스트넷(Sepolia 등) 또는 로컬 네트워크가 선택되어 있는지 확인합니다.
   - **Deploy** 버튼을 눌러 컨트랙트를 배포합니다.
5. 배포가 완료되면 하단의 **Deployed Contracts** 영역에 컨트랙트가 나타납니다. 여기서 **컨트랙트 주소 복사 아이콘**을 클릭해 주소를 복사합니다.

## 2. 프론트엔드 컨트랙트 주소 연동
1. 로컬 프로젝트 폴더 안의 `app.js` 파일을 텍스트 편집기나 VS Code 등으로 엽니다.
2. 파일 최상단(4번째 줄 부근)에 있는 `CONTRACT_ADDRESS` 변수 값에 방금 복사한 컨트랙트 주소를 붙여넣고 저장합니다.
   ```javascript
   const CONTRACT_ADDRESS = "0x방금복사한주소"; 
   ```

## 3. 웹사이트 실행 및 그룹 시연 시나리오
메타마스크 지갑 연동을 원활하게 하기 위해, 로컬 서버를 통해 여는 것을 강력히 권장합니다.
- **방법 1 (권장):** VS Code에서 프로젝트를 열고 우측 하단의 **'Go Live' (Live Server 익스텐션)** 버튼을 클릭합니다.
- **방법 2:** 터미널을 열고 프로젝트 폴더로 이동한 뒤 `python3 -m http.server 8080`을 입력하고 브라우저에서 `http://localhost:8080`에 접속합니다.

### 🌟 시연 시나리오 가이드
1. **지갑 연결:** 우측 상단의 **[지갑 연결]** 버튼을 눌러 메타마스크를 연동합니다.
2. **테스트 토큰 수령:** 대시보드 하단의 **[테스트 토큰 받기]** 버튼을 눌러 시연용 E2S 토큰(1,000개)을 받습니다.
3. **스터디 방 생성 및 참가 (가스비 최적화):**
   - **[새 방 만들기]** 버튼을 누르면 랜덤한 방 코드(예: `ST-XXXX`)가 자동 생성되며 즉시 대기실로 입장합니다. (트랜잭션 없음)
   - 브라우저의 다른 계정으로 접속하여 해당 코드를 입력하고 **[기존 방 참가]**를 누르면, 두 사용자가 서로 "대기 중 👀" 상태인 것을 현황판에서 실시간으로 확인할 수 있습니다.
4. **목표 설정 및 예치 (단일 트랜잭션):**
   - 예치할 토큰 양과 공부 시간을 입력하고 **[예치하고 공부 시작!]** 버튼을 누릅니다.
   - 내부 `_transfer` 로직 최적화로 인해 별도의 Approve 과정 없이 **단 1번의 트랜잭션만 승인**하면 됩니다.
5. **보상 클레임 및 페널티 시연:**
   - 타이머가 0초가 되면 **[인증 완료 및 보상 수령]** 버튼을 눌러 5% 이자와 함께 원금을 돌려받습니다. 현황판에 상태가 '성공 🎉'으로 바뀝니다.
   - 다른 계정으로는 중간에 **[공부 포기하기]**를 눌러 예치금을 몰수당하는 모습을 보여줍니다. 현황판에 '실패(몰수) 💀' 상태가 뜹니다.

## 4. 향후 업데이트 목표 (Roadmap)
현재 버전은 성공한 참여자에게 고정 비율(5%)의 이자를 새롭게 발행(Minting)하여 지급하고, 포기한 사람의 예치금은 컨트랙트에 귀속되는 형태로 구현되어 있습니다. 
향후에는 다음과 같은 **다이내믹 보상 시스템**을 도입하여 토크노믹스를 고도화할 계획입니다.

- **🔥 몰수 예치금 자동 재분배 시스템:** 목표 달성에 실패하거나 포기한 사람들의 예치금이 컨트랙트에 묶이는 대신, **목표 달성에 성공한 스터디 그룹 멤버들에게 자동으로 분배되도록 스마트 컨트랙트를 재설계**합니다. 이를 통해 사용자들은 서로의 실패가 나의 더 큰 보상으로 이어짐을 인지하게 되며, 참여자들의 성취욕과 집중력을 극대화할 수 있습니다.
