# Earn To Study (E2S) - Group Edition DApp

## 1. Intro (도입)

- **프로젝트명:** Earn To Study (토큰명: E2S)
- **한 줄 소개:** 여름방학의 나태함을 이겨내기 위해 사용자의 공부 시간 및 목표를 토큰으로 예치하고 달성 시 5%의 이자와 함께 돌려받는 웹3(Web3) 목표 달성 DApp.
- **발표 목적:** 방학만 되면 흐지부지되는 학업 계획을 스마트 컨트랙트의 투명한 강제성과 경제적 인센티브를 통해 해결하는 실질적인 서비스를 제안.

## 2. Problem (문제 제기)

- **작심삼일과 의지 부족:** 사람들은 다이어트, 코딩 공부 등 목표를 세우지만 의지 부족으로 쉽게 포기하는 경향이 있음. 특히나  통제가 적은 여름방학에는 이런 현상이 훨씬 심해짐.
- **기존 타이머 앱의 한계:** 기존의 스터디 타이머나 투두(To-do) 리스트 앱은 실패하더라도 나에게 돌아오는 경제적 타격이 없기 때문에 강제성이 매우 떨어짐.

## 3. Solution (해결 방안)

- **목표 설정 및 예치 (Staking):** 사용자가 하루 목표 공부 시간이나 학습 분량을 설정하고, 일정량의 토큰을 스마트 컨트랙트에 예치할 수 있도록 함.
- **확실한 성공 보상:** 타이머 종료 및 목표 달성 인증 시, 원금과 함께 보상으로 자체 발행한 ERC-20 토큰(SUCCESS Token)을 지급. 또한 예치한 금액의 5%의 토큰을 추가 보상으로 설정함.
- **실패 패널티 및 재분배:** 목표 달성에 실패한 사람들의 예치금은 성공한 사람들에게 분배되도록 스마트 컨트랙트를 설계하여 참여자들의 성취욕을 극대화함.

## 4. Stack (스택 상세 설명)

- **블록체인 네트워크:** 이더리움 테스트넷 (데모 시연 시 가스비 부담을 없애기 위해 사용)
- **스마트 컨트랙트 (Solidity):** 이더리움 블록체인 네트워크에서 발행되는 토큰의 표준 규격인 ERC-20을 기반으로 개발해. 예치(Stake), 타이머 종료 후 인증(Verify), 보상 반환(Claim) 로직을 중개자 없이 투명하게 구현할 수 있음.
- **프론트엔드 (React.js 또는 HTML/JS):** 목표 시간을 설정하고 구동되는 타이머 화면, 메타마스크(Metamask) 지갑 연결 버튼, 현재 예치된 토큰을 보여주는 대시보드를 만듦.

## 4. 향후 업데이트 목표 (Roadmap)
현재 버전은 성공한 참여자에게 고정 비율(5%)의 이자를 새롭게 발행(Minting)하여 지급하고, 포기한 사람의 예치금은 컨트랙트에 귀속되는 형태로 구현되어 있음. 
향후에는 다음과 같은 **다이내믹 보상 시스템**을 도입하여 토크노믹스를 고도화할 계획.

- **몰수 예치금 자동 재분배 시스템:** 목표 달성에 실패하거나 포기한 사람들의 예치금이 컨트랙트에 묶이는 대신, **목표 달성에 성공한 스터디 그룹 멤버들에게 자동으로 분배되도록 스마트 컨트랙트를 재설계**. 이를 통해 사용자들은 서로의 실패가 나의 더 큰 보상으로 이어짐을 인지하게 되며, 참여자들의 성취욕과 집중력을 극대화할 수 있음.
